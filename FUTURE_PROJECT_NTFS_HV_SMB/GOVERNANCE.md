# Governance

## Purpose
Define how this future-project hub should be used.

## Rules
1. `README.md` is the project entry point.
2. `MEGAPLAN_WINDOWS_PARITY.md` is the top-level strategy document.
3. `IMPLEMENTATION_BOARD.md` is the cross-program execution board.
4. Program-specific execution details live in Program A and Program B documents.
5. Architecture decisions must be recorded in `DECISION_LOG.md`.
6. Support claims must align with `PROGRAM_B_RELEASE_CLAIMS.md` and the lab matrix.
7. Risks must be updated in `RISK_REGISTER.md` when scope or claims change.

## Change Control
A change is considered material if it changes:
1. support claims
2. architecture direction
3. staffing assumptions
4. milestone boundaries
5. validated matrix assumptions

Material changes must update:
1. `DECISION_LOG.md`
2. `README.md` if reading order or file roles change
3. whichever roadmap or board document is affected
